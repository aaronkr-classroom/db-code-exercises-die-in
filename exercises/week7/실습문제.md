### 2. 개체 후보 검토
| No. | 개체 후보 | 데이터 수 | 주요 속성 | 비고 | 개체 타입 선정 여부 |
| :-: | :-: | :--- | :--- | :--- | :-: |
| 1 | 회사 | 1개 | 없음 | 데이터가 하나뿐이므로 개체에서 제외 | X |
| 2 | 게임 | 다수 | 게임 번호, 게임명, 버전 등 | 요구조건에 부합하는 주요 정보 | O |
| 3 | 회원 | 다수 | id, pw, 이름, 연락처 등 | 요구조건에 부합하는 주요 정보 | O |
| 4 | 이력 | 다수 | 시작 일시, 종료 일시, 점수 | 독립된 개체보다는 회원과 게임 간의 '관계'로 처리하는 것이 더 적합함 | X (관계로 전환) |

### 9. ER 다이어그램 작성 전 체크리스트
| 분류 | No. | 내용 | 확인 | 
| :-: | :-: | :--- | :-: |
| **개체** | 1 | 실제 데이터가 1개만 존재하는 개체가 식별되었는가? | O (회사 개체 제외함) |
| | 2 | 속성이 1개인 개체가 식별되었는가? | X (없음) |
| | 3 | 개체 이름이 중복되었는가? | X (중복 없음) |
| | 4 | 각 개체의 기본키가 결정되었는가? | O (회원: id, 게임: 게임 번호) |
| | 5 | 각 개체를 설명하는 일반 속성이 모두 식별되었는가? | O |
| | 6 | 개체 가운데 관계로 표현하면 더 적합한 것이 있는가? | O ('이력' 후보를 관계로 전환함) |
| **관계** | 1 | 관계를 설명하는 설명자(일반 속성)가 모두 식별되었는가? | O (시작/종료 일시, 점수) |
| | 2 | 관계 이름이 동사로 표현되었는가? | O ('플레이하다') |
| | 3 | 관계 이름이 현재 시제로 표현되었는가? | O |
| | 4 | 관계 이름이 개체 간의 관련성을 명확히 표현하고 있는가? | O |
| | 5 | 관계 이름이 중복되었는가? | X |
| | 6 | 관계 유형을 정확히 결정하였는가? | O (M:N) |
| **속성** | 1 | 속성 이름에 개체나 관계 이름이 포함되었는가? | △ ('게임 번호', '게임명' 등에 포함되어 있어 향후 '번호', '이름' 등으로 수정 고려 가능) |
| | 2 | 속성 이름으로 의미가 불분명한 약어가 사용되었는가? | △ ('pw'는 '비밀번호'로 수정 권장) |
| | 3 | 속성의 성격을 명확히 인식하기에 쉬운 이름이 부여되었는가? | O |
| | 4 | 속성 이름을 부여하는 데 일정한 규칙이 있는가? | O |
