# 데이터 요구 분석 명세서

## 고객 관리
### 데이터 요구 사항
고객: 고객 정보를 관리하기 위한 개체
- 유지 속성: 고객 ID, 이름, 주민등록번호, 휴대폰 번호, 주소, 고객 등급, 여행상품 참여 횟수
- 제약/규칙: 고객 등급은 '일반', '우수', '최우수'로만 나뉨

### 트랜잭션 요구 사항
- 고객 정보 관리: 신규 고객 정보 등록, 수정, 삭제 
- 고객 등급 갱신: 고객의 누적 여행상품 참여 횟수를 조회하여 등급을 자동 산정 및 업데이트 (6회 이상: 최우수, 3~5회: 우수, 그 외: 일반)
- 고객 조회: 조건(ID, 이름, 등급 등)에 따른 고객 정보 검색 기능

## 여행상품 관리
### 데이터 요구 사항
여행상품: 판매하는 여행 프로그램 정보를 관리하기 위한 개체
- 유지 속성: 여행번호, 출발지, 도착지, 프로그램 정보, 시작일시, 종료일시, 최소 출발인원, 최대 인원, 여행경비, 예약금

관광버스: 보유 중인 20대의 버스 자원 개체
- 유지 속성: 차량번호, 좌석 수, 출고년도

### 트랜잭션 요구 사항
- 상품 정보 관리: 신규 여행상품 개발(등록), 수정, 삭제 기능 
- 상품 조회: 현재 판매 중인 계절별/상태별 여행상품 목록 및 세부 정보 검색 기능 
- 출발 여부 결정: 여행 시작일 기준 2주 전 시점에 예약 인원(최소 출발인원 충족 여부 등)을 파악하여 출발 여부를 확정하는 기능 
- 배차 및 자원 할당: 여행상품 출발이 확정되면 관광버스 1대와 운전기사 1명을 해당 상품에 매핑(배정)하는 기능 
- 담당자 배정: 각 여행상품에 1명의 담당 직원(사번)을 매핑하는 기능 (1명의 직원은 여러 상품 담당 가능)

## 예약 관리
### 데이터 요구사항
예약: 고객의 여행상품 예약 내역을 관리하는 개체
- 유지 속성: 예약번호(고유식별자), 예약일자, 예약금 납부여부, 잔금 납부여부, 예약상태(예약대기, 예약완료, 취소 등)
- 관계 속성: 어떤 고객(고객ID)이 어떤 상품(여행번호)을 예약했는지 연결

### 트랜잭션 요구사항
- 예약 등록 및 결제: 홈페이지나 앱을 통해 고객이 예약을 생성하고 결제를 진행하는 기능
- 예약금 미납 취소 처리: 예약일시 기준 24시간 이내에 예약금 결제가 이루어지지 않은 내역을 찾아 자동으로 '예약 취소' 처리하는 기능
- 잔금 미납 취소 처리: 상품 출발이 결정된 후, 1주일 이내에 잔금을 결제하지 않은 내역을 '예약 취소' 처리하는 기능 (이때 납부된 예약금은 반환 불가 처리)
- 회사 귀책 환불 처리: 여행사의 사정(인원 미달 등)으로 상품 출발이 취소된 경우, 1주일 이내에 고객에게 예약금을 반환 처리하는 기능
- 예약 상태 조회: 고객 본인의 예약 및 결제 진행 상황 확인 기능, 직원이 특정 상품의 전체 예약 현황을 확인하는 기능

## 직원 관리
### 데이터 요구사항
직원: 여행사 소속 30여 명의 정규 직원 개체
- 유지 속성: 사번, 이름, 주민등록번호, 휴대폰 번호, 급여, 담당 업무(상품관리, 예약관리, 배차, 직원관리, 고객관리 등), 입사일 

운전기사: 월 단위 계약직인 운전기사 개체
- 유지 속성: 기사ID, 이름, 주민등록번호, 연락처, 운전경력, 시급, 계약일, 계약기간

### 트랜잭션 요구사항
- 인사 정보 관리: 정규 직원 및 계약직 운전기사의 정보 등록, 수정, 삭제 기능
- 업무 권한 관리: 직원의 담당 업무에 따라 접근할 수 있는 메뉴(자신이 담당하는 여행상품 및 해당 상품의 예약 상황 등) 권한을 부여하고 조회하는 기능
